# 如何查看 HLS 自适应切换日志

## 方法 1: 主页面日志区域（推荐）

访问 **http://localhost:4000**

页面底部有"事件日志"区域，会实时显示：

```
[12:34:56] 加载 HLS 视频: /hls/知否/master.m3u8
[12:34:57] ✅ 清单解析完成 - 找到 3 个画质选项
[12:34:57] 🔄 准备切换到: 720p (2.50 Mbps)
[12:34:58] ✅ 已切换到: 720p (2.50 Mbps)
[12:34:58] ⬇️ 分片 0 下载完成: 2.13 MB, 用时 0.52s, 速度 32.50 Mbps
[12:35:04] ⬇️ 分片 1 下载完成: 2.15 MB, 用时 0.48s, 速度 35.42 Mbps
...
```

**这个日志会自动滚动，显示最新的 30 条记录。**

## 方法 2: 浏览器 Console（已添加）

1. 打开 http://localhost:4000
2. 按 **F12** 打开开发者工具
3. 切换到 **Console** 标签

你会看到彩色的日志输出：

```javascript
[12:34:56] 加载 HLS 视频: /hls/知否/master.m3u8
[12:34:57] ✅ 清单解析完成 - 找到 3 个画质选项  (绿色)
[12:34:57] 🔄 准备切换到: 720p (2.50 Mbps)
[12:34:58] ✅ 已切换到: 720p (2.50 Mbps)  (绿色)
[12:34:58] ⬇️ 分片 0 下载完成: 2.13 MB, 用时 0.52s, 速度 32.50 Mbps
```

## 方法 3: 调试页面（最详细）⭐ 推荐

访问 **http://localhost:4000/debug.html**

这是专门的调试工具，提供：

### 1. 实时指标面板
- 当前画质
- 估算带宽
- 实际下载速度
- 缓冲区大小
- 已下载分片数
- 总下载量

### 2. 带宽历史图表
- 实时绘制带宽变化曲线
- 显示 360p/720p/1080p 需求线
- 可以直观看到网速波动

### 3. 详细事件日志
- 每个分片的下载详情
- 画质切换的完整过程
- 错误和警告信息

### 4. 可用码率信息
- 显示所有可用的画质选项
- 每个画质的分辨率和码率

## 方法 4: Chrome DevTools Network 标签

1. 打开 http://localhost:4000
2. 按 **F12** → **Network** 标签
3. 播放视频

你会看到所有的网络请求：

```
Name                Type        Size      Time
----------------------------------------------
master.m3u8         xhr         245 B     12 ms
720p.m3u8           xhr         523 B     8 ms
720p_000.ts         xhr         2.1 MB    520 ms
720p_001.ts         xhr         2.2 MB    480 ms
720p_002.ts         xhr         2.0 MB    550 ms
...
```

**当画质切换时，你会看到文件名变化：**

```
720p_002.ts   ← 原来的画质
720p_003.ts
360p_004.ts   ← 切换到 360p！
360p_005.ts
360p_006.ts
```

## 如何测试自适应切换？

### 测试 1: 使用 Chrome 限速

1. 打开 http://localhost:4000
2. F12 → Network 标签
3. 点击 **Throttling** 下拉菜单

#### 步骤：

1. **选择 "No throttling"**
   ```
   → 播放器会选择 1080p 或 720p
   → 日志显示: ✅ 已切换到: 1080p
   ```

2. **切换到 "Slow 3G"**
   ```
   → 几秒后自动降低画质
   → 日志显示:
      🔄 准备切换到: 360p
      ✅ 已切换到: 360p
   → Network 标签中看到: 360p_xxx.ts
   ```

3. **切换回 "No throttling"**
   ```
   → 观察几个分片后，逐渐升级画质
   → 日志显示:
      🔄 准备切换到: 720p
      ✅ 已切换到: 720p
   ```

## 观察要点

### 1. 页面日志区域会显示

```
✅ 已切换到: 720p (2.50 Mbps)         ← 当前画质
⬇️ 分片 0 下载完成: 2.13 MB, 用时 0.52s, 速度 32.50 Mbps
                                      ↑
                              注意这个速度
```

**如果速度下降：**
```
⬇️ 分片 5 下载完成: 2.15 MB, 用时 2.5s, 速度 6.88 Mbps
                                        ↑
                                  速度变慢了！

🔄 准备切换到: 360p (0.80 Mbps)        ← 自动降级
✅ 已切换到: 360p (0.80 Mbps)
```

### 2. Console 中会输出

带颜色的日志，方便区分：
- 🟢 绿色 = 成功（清单解析、画质切换成功）
- 🟡 黄色 = 警告（准备切换）
- 🔴 红色 = 错误
- ⚪ 白色 = 普通信息（分片下载）

### 3. debug.html 中的图表

会实时绘制：
- 蓝色线 = 估算带宽
- 绿色线 = 实际下载速度
- 虚线 = 各画质需求（0.8 Mbps, 2.5 Mbps, 5.0 Mbps）

**当蓝色线下降并穿过虚线时，就会触发降级！**

## 完整测试流程

```bash
# 1. 确保服务器运行
cd hls-streaming
npm start

# 2. 打开浏览器
# http://localhost:4000

# 3. 打开 Chrome DevTools
# F12 → Console 标签 + Network 标签（同时看）

# 4. 播放视频
# 点击视频列表中的"知否"

# 5. 观察日志
# Console: 看到分片下载日志
# 页面底部: 看到事件日志
# Network: 看到 .ts 文件请求

# 6. 测试限速
# Network → Throttling → "Slow 3G"
# 等待 5-10 秒，观察画质切换
```

## 日志类型说明

### 主页面和 Console 的日志

| 日志 | 含义 |
|------|------|
| `加载 HLS 视频: ...` | 开始加载视频 |
| `✅ 清单解析完成 - 找到 3 个画质选项` | 成功解析 master.m3u8 |
| `🔄 准备切换到: 720p` | hls.js 决定切换画质 |
| `✅ 已切换到: 720p` | 画质切换完成 |
| `⬇️ 分片 X 下载完成: ...` | 每个视频分片下载完成 |
| `❌ 致命错误: ...` | 发生错误 |

### debug.html 的额外日志

- 每个分片的详细统计
- 估算带宽的实时值
- 缓冲区状态变化
- 丢帧统计

## 常见问题

### Q1: 页面日志区域没有内容？

**检查：**
1. 页面是否完全加载？
2. 是否选择了视频？
3. 滚动到页面底部，日志区域在"实时统计"下方

### Q2: Console 中没有日志？

**检查：**
1. 是否按了 F12？
2. 是否切换到 Console 标签？
3. 是否有其他错误阻止了脚本运行？

### Q3: 没有看到画质切换？

**原因：**
- 网络速度一直很快 → 一直保持高画质
- 网络速度一直很慢 → 一直保持低画质

**解决：**
使用 Chrome DevTools Network Throttling 手动限速

### Q4: 想看更详细的日志？

访问 **http://localhost:4000/debug.html**

这个页面提供最详细的调试信息，包括：
- 实时带宽图表
- 每个分片的详细统计
- hls.js 的所有事件
