## 快速开始（3 步）

### 1. 安装依赖

```bash
cd hls-streaming
npm install
```

### 2. 转码视频

#### 方法 1: 一键转码所有视频（推荐）

```bash
npm run transcode-all
```

这会自动转码 `video-streaming/videos/` 中的所有视频。

#### 方法 2: 手动转码单个视频

```bash
# 转码"知否.mov"
node transcode.js ../video-streaming/videos/知否.mov

# 转码"社火.mov"
node transcode.js ../video-streaming/videos/社火.mov
```

## 测试网络自适应

### 在 Chrome 中测试

1. 打开 http://localhost:4000
2. 按 F12 打开开发者工具
3. 切换到 **Network** 标签
4. 点击 **Throttling** 下拉菜单

#### 测试场景 1: 慢速网络

```
Throttling 选择: Slow 3G
       ↓
播放器自动选择: 360p
       ↓
观察: 流畅播放，画质较低
```

#### 测试场景 2: 快速网络

```
Throttling 选择: No throttling
       ↓
播放器自动选择: 1080p
       ↓
观察: 高清播放
```

#### 测试场景 3: 网速变化

```
开始: Fast 3G → 720p
       ↓ (切换到 Slow 3G)
自动降低: 720p → 360p
       ↓ (切换回 Fast 3G)
自动提高: 360p → 720p
```

### 观察 Network 请求

你会看到这样的请求序列：

```
1. master.m3u8           # 获取主播放列表
2. 720p.m3u8            # 选择 720p (假设网速适中)
3. 720p_000.ts          # 下载第一个分片
4. 720p_001.ts          # 下载第二个分片
5. 720p_002.ts          # 下载第三个分片
...

(如果网速变慢，自动切换)
10. 360p.m3u8           # 切换到 360p 播放列表
11. 360p_005.ts         # 下载 360p 分片
```

## 文件大小参考

原始视频 → 转码后 HLS 文件大小对比：

| 视频 | 原始大小 | 360p | 720p | 1080p | 总计 |
|------|---------|------|------|-------|------|
| 知否.mov | 72.6 MB | ~15 MB | ~30 MB | ~50 MB | ~95 MB |
| 社火.mov | 102.7 MB | ~20 MB | ~40 MB | ~70 MB | ~130 MB |

注意：
- HLS 总文件大小会比原文件大（因为有 3 个码率）
- 但用户只下载他们需要的码率
- 慢速网络用户只下载 360p，节省流量

## 故障排除

### 问题 1: FFmpeg 未找到

```bash
# 检查 FFmpeg 是否安装
ffmpeg -version

# 如果没安装
brew install ffmpeg
```

### 问题 2: 转码失败

```
Error: ffmpeg exited with code 1
```

**解决方案：**
```bash
# 检查视频文件是否损坏
ffmpeg -i ../video-streaming/videos/知否.mov

# 如果编码不兼容，先转换
ffmpeg -i 知否.mov -c:v libx264 -c:a aac 知否_h264.mov
```

### 问题 3: 端口 4000 被占用

修改 `hls-server.js`:

```javascript
const PORT = 5000;  // 改为其他端口
```
