Range Requests 自动行为演示

测试 1: preload="metadata" (默认行为)

预期行为: 页面加载时，浏览器自动发送一个小的 Range 请求（约 32KB），只获取视频元数据。
打开 DevTools → Network 查看请求头: Range: bytes=0-32767

测试 2: preload="auto"

预期行为: 浏览器会预加载更多数据（1-3MB），准备流畅播放。
你会看到多个连续的 Range 请求。

测试 3: preload="none"

预期行为: 页面加载时不发送任何请求，直到你点击播放按钮。
观察 Network 标签，只有点击播放后才会出现请求。

观察要点: 点击按钮跳转时，观察 DevTools Network 标签。
每次跳转，浏览器会自动计算字节位置并发送新的 Range 请求！
例如跳到 50%，你会看到 Range: bytes=5242880-...

关键发现

  • 浏览器完全自动处理 Range 请求 - 不需要任何 JavaScript
  • 拖动进度条会触发新的 Range 请求 - 直接跳到目标位置
  • 服务器返回 206 状态码 - 表示部分内容
  • Content-Range 头告诉浏览器 - 这是哪一段数据
  • 边下载边播放 - 无需等待完整视频
